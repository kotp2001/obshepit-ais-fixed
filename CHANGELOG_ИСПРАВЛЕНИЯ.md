# Исправления АИС «Общепит»

Ниже — что и почему изменено по файлам с ошибок 8, 9 и ТЗ. Диплом и ТЗ диплома не трогались.

## Главная причина большинства багов

В нескольких шаблонах HTML/JS/CSS по ошибке оказались **внутри блока `{% block title %}`**.
Из-за этого они попадали в тег `<title>` и просто не выполнялись — код был написан, но не работал.
Плюс пара ошибок экранирования кавычек в JavaScript. Исправлено.

## Бэкенд

### `restaurant/models.py`
- `Order.created_at` теперь `default=timezone.now` (было: без значения по умолчанию).
  Это устраняет **IntegrityError «null value in column created_at»** (ошибки 9, п.5).
  Причина была в рассинхроне: в первой миграции поле было `auto_now_add=True` (NOT NULL),
  а в модели его сменили на `null=True` без миграции.

### `restaurant/migrations/0004_order_created_at_default.py` (новый)
- Точечная миграция, приводящая поле `created_at` в соответствие с моделью.
  Сделана узкой намеренно, чтобы не задеть боевую PostgreSQL-схему.

### `restaurant/migrations/0005_maintenance_log_data.py` (новый)
- Заполняет журнал ТО 12 записями: исполнитель **«Дубровин КС»**, период с 1 по 20 мая 2026,
  интервал 1–2 дня (ошибки 9, п.4). Миграция идемпотентна (повторно не дублирует) и обратима.

### `restaurant/views.py`
- `api_create_order`: заказ теперь привязывается к текущему сотруднику (`order.waiter = request.user`).
- `api_active_orders`: имя/роль сотрудника считаются корректно — администратор показывается как
  **`admin` (Администратор)**, а не «none»; для официанта/повара — имя и роль (ошибки 8, п.5).

### `restaurant/admin.py`
- `LoginAttemptAdmin`: добавлено действие **«Снять блокировку (сбросить 15-минутное ожидание)»**
  и колонка «Заблокирован» (ошибки 8, п.6).
- `OrderAdmin`: убрано поле суммы из формы — сумма считается **автоматически** по позициям заказа
  (метод `save_related`), плюс дата/время создания берутся из календаря и часов (ошибки 9, п.1).

## Как развернуть

```bash
# 1. Применить изменения (если используете патч)
git apply obshepit_fixes.patch
# либо просто скопировать изменённые файлы поверх своих

# 2. Зафиксировать и отправить
git add -A
git commit -m "Исправление багов: фильтры, сумма заказа, блокировка, журнал ТО, IntegrityError"
git push

# 3. На Render выполнить миграции (важно для 0004 и 0005)
python manage.py migrate

# 4. Manual Deploy в панели Render
```

  Если есть точные коды цветов — пришлите, подставлю.
